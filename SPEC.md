# Rhizome（灵犀树）

## 1. 项目概述

**项目名称**: Rhizome（灵犀树）

**项目类型**: 基于LangChain的AI智能体应用

**核心功能**: 通过对话方式帮助用户建立、管理和扩充个人知识体系。Agent在回答用户问题的同时，自动将知识整理到结构化的知识目录中，实现知识的积累和系统化。

**目标用户**: 想要系统化学习、构建个人知识体系的用户

## 2. 技术架构

### 技术栈
- **LLM框架**: LangChain
- **大模型**: OpenAI GPT系列, Anthropic Claude, Ollama, Azure OpenAI (可扩展支持其他模型)
- **向量存储**: Chroma (本地向量数据库)
- **前端**: Gradio (快速构建对话界面)
- **数据存储**: JSON/SQLite (知识目录和元数据)

### 支持的LLM供应商
| 供应商 | 环境变量配置 |
|--------|-------------|
| OpenAI | `LLM_PROVIDER=openai`, `OPENAI_API_KEY`, `OPENAI_MODEL` |
| Anthropic | `LLM_PROVIDER=anthropic`, `ANTHROPIC_API_KEY`, `OPENAI_MODEL` |
| Ollama | `LLM_PROVIDER=ollama`, `OPENAI_API_BASE=http://localhost:11434`, `OPENAI_MODEL` |
| Azure OpenAI | `LLM_PROVIDER=azure`, `OPENAI_API_KEY`, `OPENAI_API_BASE`, `OPENAI_MODEL` |

## 3. 功能规格

### 3.1 核心功能

#### F1: 智能问答
- 用户通过自然语言提问
- Agent理解问题意图
- 从知识库检索相关信息
- 生成准确、上下文相关的回答

#### F2: 知识目录管理
- 自动创建知识分类目录
- 支持多级目录结构
- 目录节点包含: 主题名称、关键词、相关问题、答案摘要
- 用户可查看、编辑知识目录

#### F3: 知识积累与扩展
- Agent回答问题时自动判断所属知识类别
- 同一问题的多次讨论可自动关联
- 支持知识的细粒度更新而非全量覆盖

#### F4: 智能检索优化
- **目录索引**: 预先建立知识主题目录，只检索相关目录下的知识
- **关键词匹配**: 问题关键词与目录节点匹配
- **向量检索**: 对目录下的知识进行语义相似度检索
- **混合检索**: 结合关键词和向量检索，提升准确性

### 3.2 数据结构设计

#### 知识目录结构 (Knowledge Catalog)
```json
{
  "id": "uuid",
  "name": "主题名称",
  "keywords": ["关键词1", "关键词2"],
  "parent_id": "父目录ID或null",
  "children": ["子目录ID列表"],
  "knowledge_items": ["知识条目ID列表"],
  "created_at": "创建时间",
  "updated_at": "更新时间"
}
```

#### 知识条目结构 (Knowledge Item)
```json
{
  "id": "uuid",
  "catalog_id": "所属目录ID",
  "question": "用户问题",
  "answer": "答案内容",
  "keywords": ["问题关键词"],
  "embedding": "向量嵌入",
  "sources": ["参考来源"],
  "confidence": 0.95,
  "created_at": "创建时间",
  "updated_at": "更新时间"
}
```

### 3.3 用户交互流程

```
用户提问 → Agent理解 → 目录匹配 → 知识检索 → 生成回答 → 更新知识库
                           ↓
                     [如无匹配目录]
                           ↓
                     创建新知识目录
```

## 4. 模块设计

### 4.1 Agent模块 (agent/)
- `base_agent.py`: Agent基类，定义通用接口
- `qa_agent.py`: 问答Agent实现
- `prompt_templates.py`: 提示词模板

### 4.2 知识管理模块 (knowledge/)
- `catalog_manager.py`: 知识目录管理
- `knowledge_store.py`: 知识存储与检索
- `embedding_manager.py`: 向量嵌入管理

### 4.3 存储模块 (storage/)
- `json_storage.py`: JSON文件存储
- `vector_store.py`: Chroma向量存储封装

### 4.4 界面模块 (ui/)
- `gradio_app.py`: Gradio对话界面

## 5. 验收标准

### 功能验收
- [ ] 用户可以向Agent提问并获得回答
- [ ] Agent能够自动将知识分类到目录
- [ ] 用户可以查看知识目录结构
- [ ] 知识检索只查看相关目录下的知识
- [ ] 多次提问同一主题会自动关联

### 性能验收
- [ ] 首次响应时间 < 5秒
- [ ] 知识检索时间 < 1秒
- [ ] 支持1000+知识条目的存储和检索

### 可扩展性
- [ ] 易于更换底层LLM模型
- [ ] 支持自定义知识目录结构
- [ ] 支持导出/导入知识数据

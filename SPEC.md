# Rhizome（灵犀树）

## 1. 项目概述

**项目名称**: Rhizome（灵犀树）

**项目类型**: 基于 LangChain 的 AI 智能体应用

**核心功能**: 通过对话方式帮助用户建立、管理和扩充个人知识体系。Agent 在回答用户问题的同时，自动将知识整理到结构化的知识目录中，实现知识的积累和系统化。

**目标用户**: 想要系统化学习、构建个人知识体系的用户

## 2. 技术架构

### 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| LLM 框架 | LangChain | 统一的大模型接口 |
| 大模型 | OpenAI GPT 等 | 可插拔式更换 |
| 嵌入模型 | HuggingFace | 本地缓存优化，支持离线使用 |
| 向量数据库 | Chroma | 本地持久化向量存储 |
| 后端框架 | FastAPI | 高性能异步 API 框架 |
| 前端框架 | Vue 3 | 响应式前端框架 |
| 构建工具 | Vite | 快速的前端构建工具 |
| 实时通信 | SSE | Server-Sent Events 流式输出 |
| 环境管理 | uv | 现代化的 Python 包管理 |

### 支持的 LLM 供应商

| 供应商 | 环境变量配置 |
|--------|-------------|
| OpenAI | `LLM_PROVIDER=openai`, `OPENAI_API_KEY`, `OPENAI_MODEL` |
| Anthropic | `LLM_PROVIDER=anthropic`, `ANTHROPIC_API_KEY`, `OPENAI_MODEL` |
| Ollama | `LLM_PROVIDER=ollama`, `OPENAI_API_BASE=http://localhost:11434`, `OPENAI_MODEL` |
| Azure OpenAI | `LLM_PROVIDER=azure`, `OPENAI_API_KEY`, `OPENAI_API_BASE`, `OPENAI_MODEL` |

### 支持的嵌入模型

| 类型 | EMBEDDING_PROVIDER | 模型示例 | 说明 |
|------|-------------------|----------|------|
| 本地模型 | `local` | BAAI/bge-small-zh-v1.5 | HuggingFace 本地模型，离线可用 |
| OpenAI | `openai` | text-embedding-ada-002 | OpenAI 嵌入 API |

## 3. 功能规格

### 3.1 核心功能

#### F1: 智能问答
- 用户通过自然语言提问
- Agent 理解问题意图
- 从知识库检索相关信息
- 生成准确、上下文相关的回答
- **流式输出**: 实时返回回答内容，提升用户体验

#### F2: 知识目录管理
- 自动创建知识分类目录
- 支持多级目录结构
- 目录节点包含: 主题名称、关键词、相关问题、答案摘要
- 用户可查看、编辑知识目录
- **可视化展示**: 树形组件展示目录结构

#### F3: 知识积累与扩展
- Agent 回答问题时自动判断所属知识类别
- 同一问题的多次讨论可自动关联
- 支持知识的细粒度更新而非全量覆盖
- **后台异步存储**: 不阻塞用户交互

#### F4: 智能检索优化
- **目录索引**: 预先建立知识主题目录，只检索相关目录下的知识
- **关键词匹配**: 问题关键词与目录节点匹配
- **向量检索**: 对目录下的知识进行语义相似度检索
- **混合检索**: 结合关键词和向量检索，提升准确性

#### F5: 对话历史
- 保存对话历史记录
- 支持查看历史对话
- 自动加载最近对话

#### F6: 知识详情查看
- 点击知识条目查看完整内容
- **Markdown 渲染**: 支持标题、列表、代码块、表格等格式
- 显示关键词、创建时间等元数据

#### F7: 统计分析
- 总知识条目、目录数统计
- 今日/本周/本月新增统计
- 最近添加的知识列表
- 目录分布可视化
- 热门关键词云

### 3.2 数据结构设计

#### 知识目录结构 (Knowledge Catalog)
```json
{
  "id": "uuid",
  "name": "主题名称",
  "keywords": ["关键词1", "关键词2"],
  "parent_id": "父目录ID或null",
  "children": ["子目录ID列表"],
  "knowledge_items": ["知识条目ID列表"],
  "created_at": "创建时间",
  "updated_at": "更新时间"
}
```

#### 知识条目结构 (Knowledge Item)
```json
{
  "id": "uuid",
  "catalog_id": "所属目录ID",
  "question": "用户问题",
  "answer": "答案内容（支持Markdown）",
  "keywords": ["问题关键词"],
  "sources": ["参考来源"],
  "confidence": 0.95,
  "created_at": "创建时间",
  "updated_at": "更新时间"
}
```

#### 对话历史结构 (Chat History)
```json
{
  "id": "uuid",
  "question": "用户问题",
  "answer": "AI回答",
  "catalog_id": "所属目录ID",
  "created_at": "创建时间"
}
```

### 3.3 用户交互流程

```
用户提问 → Agent理解 → 目录匹配 → 知识检索 → 流式生成回答 → 后台更新知识库
                           ↓
                     [如无匹配目录]
                           ↓
                     创建新知识目录
```

## 4. 模块设计

### 4.1 Agent 模块 (agent/)
- `qa_agent.py`: 问答 Agent 实现，支持流式输出和依赖注入
- `prompt_templates.py`: 提示词模板

### 4.2 知识管理模块 (knowledge/)
- `catalog_manager.py`: 知识目录管理
- `knowledge_store.py`: 知识存储与检索

### 4.3 存储模块 (storage/)
- `json_storage.py`: JSON 文件存储
- `vector_store.py`: Chroma 向量存储封装

### 4.4 后端模块 (backend/)
- `main.py`: FastAPI 应用入口
- `dependencies.py`: 依赖注入管理
- `routes/chat.py`: 对话接口（支持 SSE 流式）
- `routes/knowledge.py`: 知识管理接口
- `routes/catalog.py`: 目录管理接口

### 4.5 前端模块 (frontend/)
- `views/ChatView.vue`: 对话页面
- `views/CatalogView.vue`: 知识目录页面
- `views/SearchView.vue`: 搜索页面
- `views/StatsView.vue`: 统计页面
- `components/TreeNode.vue`: 目录树节点组件
- `api.js`: API 封装

## 5. API 接口设计

### 5.1 对话接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/chat` | 发送消息并获取回复 |
| POST | `/api/chat/stream` | 流式对话（SSE） |
| GET | `/api/chat/history` | 获取对话历史 |

### 5.2 知识管理接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/knowledge` | 获取所有知识 |
| GET | `/api/knowledge/statistics` | 获取统计数据 |
| GET | `/api/knowledge/search` | 搜索知识 |
| GET | `/api/knowledge/catalog/{id}` | 获取目录下的知识 |
| DELETE | `/api/knowledge/{id}` | 删除知识 |

### 5.3 目录管理接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/catalog/tree` | 获取目录树 |
| GET | `/api/catalog` | 获取所有目录 |
| POST | `/api/catalog` | 创建目录 |
| DELETE | `/api/catalog/{id}` | 删除目录 |

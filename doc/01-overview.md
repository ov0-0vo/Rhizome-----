# 1. 项目概述

## 1.1 项目简介

Rhizome（灵犀树）是一个基于 LangChain 的知识体系智能体项目，旨在帮助用户通过对话方式建立、管理和扩充个人知识体系。

**核心理念**：像培育一棵树一样培养知识体系——从种子（问题）开始，逐步长出枝干（知识目录）和叶子（具体知识）。

## 1.2 核心功能

### 1.2.1 智能问答

用户可以随时向 Agent 提问，无论是关于编程、科学、历史还是任何其他领域的问题。Agent 会：

1. 理解问题的意图和关键信息
2. 从已有知识库中检索相关信息
3. 结合上下文生成准确、连贯的回答
4. 自动将新知识整理到知识体系中

### 1.2.2 知识目录管理

系统自动维护一个树状结构的知识目录：

- **自动分类**：Agent 在回答问题时自动判断问题所属的知识领域
- **层级结构**：支持多级目录，满足复杂知识体系的组织需求
- **动态扩展**：随着用户的不断使用，知识目录自然扩展

### 1.2.3 智能检索优化

为了解决知识数据增长带来的检索效率问题，系统采用：

- **目录索引预筛选**：问题首先匹配到具体知识目录
- **向量检索**：在相关目录下进行语义相似度检索
- **混合检索策略**：结合关键词匹配和向量相似度

### 1.2.4 多模型支持

支持多种大语言模型供应商：

| 供应商 | 模型示例 | 适用场景 |
|--------|----------|----------|
| OpenAI | GPT-3.5, GPT-4 | 通用场景，高质量输出 |
| Anthropic | Claude 3 | 长文本处理，安全性要求高 |
| Ollama | Llama 2, Mistral | 本地部署，隐私敏感 |
| Azure OpenAI | GPT-4 (Azure) | 企业用户，合规要求 |

## 1.3 设计原则

### 1.3.1 用户体验优先

- **零配置启动**：安装后即可使用默认配置运行
- **渐进式学习**：系统从用户的提问中学习，无需手动录入知识
- **透明可解释**：用户可以查看知识如何被组织和分类

### 1.3.2 开发者友好

- **模块化设计**：各组件独立，可单独测试和替换
- **清晰接口**：通过 LangChain 标准化接口，易于扩展
- **完整文档**：提供从入门到精通的完整技术文档

### 1.3.3 性能与可扩展性

- **本地向量存储**：使用 Chroma，无需额外服务
- **按需加载**：只加载当前对话相关的知识
- **目录隔离检索**：避免扫描整个知识库

## 1.4 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| LLM 框架 | LangChain | 统一的大模型接口 |
| 大模型 | OpenAI GPT 等 | 可插拔式更换 |
| 嵌入模型 | HuggingFace | 本地缓存优化，支持离线使用 |
| 向量数据库 | Chroma | 本地持久化向量存储 |
| 前端界面 | Gradio | 快速构建 ML UI |
| 环境管理 | uv | 现代化的 Python 包管理 |

## 1.5 适用场景

- **个人知识管理**：建立个人知识库，告别遗忘
- **学习助手**：辅助学习新领域，系统化整理笔记
- **研究辅助**：管理研究资料，快速检索相关信息
- **团队知识库**：共享知识资产，沉淀组织经验
